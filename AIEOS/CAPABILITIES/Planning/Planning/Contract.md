# Implementation Planning Capability Contract

## Core Purpose
Analyze requirements, map out tasks, identify dependencies, and manage risks.

## Quality Gates
- **Entry Preconditions**: Clear, written feature requirements statement
- **Required Context**: System architecture maps and existing task boards
- **Execution Instructions**: Trace affected modules, map variables, and write step-by-step checklists
- **Verification Assertions**: Cross-check that new task changes do not conflict with open tasks
- **Exit Requirements**: Structured implementation plan markdown document
