# Cognitive Risk Analysis Contract

## Core Purpose
Map operational hazards, downside exposures, and single points of failure.

## Quality Gates
- **Entry Preconditions**: Plan specifications and boundary context parameters
- **Required Context**: Historically observed project outages and failure logs
- **Execution Instructions**: Isolate variables, calculate likelihood vs severity, and assign owners
- **Verification Assertions**: Every severity index > 7 hazard contains an approved mitigation pathway
- **Exit Requirements**: Risk registry brief
